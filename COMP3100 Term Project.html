CS3100 Project A and B

The 3100 project requires the development of a web application that stores spreadsheets for users, these spreadsheets can be shared between users of the web site. The project will be done in two parts, A and B. Part A will implement user account management for authentication and authorization and will provide mock-ups of the web pages for part B. Authentication identifies a user, normally with a password. Authorization determines what operations and access a particular user has on the web applications. In part A, there should be normal users and administrative users. The mock-ups will describe the features to be implemented in part B.

Some of the possible features of the web application include the following. The web application should allow for the creation of a spreadsheet with growable rows and columns. Data can be entered into the cells of the spreadsheet. The cells should be editable. The spreadsheet should be able to perform simple row and column formulas (e.g., sum, min, max, avg ...). The spreadsheet should be shareable to other users on the web site, or it should be private. The shared spreadsheets should be copyable. The web application should be able to draw simple plots of the data in the spreadsheet.

The general requirements for the project are:

  • A web application includes the HTML, CSS, JavaScript, media files, and any other files used by the browsers and server.
  • The nodejs and the express framework must be used. Any node packages can be used except ones that implement the core features of a spreadsheet. The project should use npm for building and testing. The Yarn package manager can be used instead of npm.
  • CSS must be used to style the pages, the styling should be consistent across all pages.
  • Communication with JSON must be used for some of the pages (features).
  • Sqlite must be used as the database engine for any database access. No other database system is allowed.
  • Web components should be used for part of the application. The spreadsheet should be implemented with a web component.
  • Web sockets can be used to implement client/server communication.
  • Part A must provide the following:
    • user authentication (i.e., the system requires users to be registered and they must sign-in to have access to the system),
    • attempting to access the web site without being logged in, should redirect the user to the login/registration page,
    • a registration web page to allow a user to create an account,
    • a web page to allow a user to change account details (e.g., change the account's password),
    • a set of administrative web pages that can modify and delete user accounts (administrative web pages should only be accessible by an administrator, this means that there is a normal role and an administrative role),
    • data for user accounts must be stored in a Sqlite database. The type of user account needs to be stored (e.g., user or admin).
    • mock up pages for the part B of the web application with a brief use case flow description. Example mock ups include a list of spreadsheets belong to a user, a sample spreadsheet, a list of spreadsheets that are shared.
  • Part B must provide at least the following features/pages:
    • a page the lists the shared spreadsheets,
    • the administrative user will have access to all the spreadsheets,
    • the shared spreadsheet should provide a button to copy a spreadsheet,
    • a page that lists the spreadsheets belong to the user (both private and sharable),
    • each spreadsheet should have a name, and whether it is sharable,
    • the listing page should have a button/form to create a new spreadsheet,
    • the listing page should be able to edit the name, delete the spreadsheet, and changes its sharing status,
    • a page to edit a spreadsheet, new rows and columns could be added, rows or columns could be deleted, cells could be editable,
    • a spreadsheet should be exported using CSV file,
    • a spreadsheet should be imported using CSV file,
      • The following two requiements have been deleted
    • any spreadsheet cell with a formula should be updated whenever the matching rows or columns change,
    • the following formulas should be implemented:
      • abs(cell), absolute value for a cell
      • sqrt(cell), square root for a cell
      • pow(cell,constant), raise cell to a power of a constant
      • max(any part row) or max(any part column)
      • min(any part row) or min(any part column)
      • avg(any part row) or avg(any part column)
      • sum any part row or column with the same number of elements

    • any changes to a spreadsheet should be saved on the server, and
    • at least three graphical reporting features (pie, line, bar graphs).
    • to achieve full marks extra features, decided by the student, have to be added to the application (e.g., admistrative features, graphical reports, ...).
